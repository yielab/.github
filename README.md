## Projects in development

### [Tack](https://github.com/yielab/tack)
*Self-hosted project manager — one binary, no Docker, no database server*

A complete project manager that ships as a single ~5 MB statically-linked binary. Your data is one SQLite file next to the binary. No containers, no Postgres, no compose file. Kanban, timeline, sprints, calendar, full-text search, a REST API, CLI with `--json`, GitHub/Linear import, and webhooks. Vocabulary reshapes per project — the same tool runs a software sprint, a kitchen renovation, or a thesis.

`Rust · Axum · SolidJS · SQLite · single binary`

---

### [Docket](https://github.com/yielab/docket)
*Cost-aware ops layer for OpenClaw agent fleets*

OpenClaw is a powerful agent framework. It has no native cost control. Docket sits on top of it and adds the ops layer: per-agent USD budget caps with auto-pause, session-based project isolation so contexts don't bleed across work, role-to-model right-sizing, and `docket doctor` for drift and health detection. Spec-driven Bash with RFC 2119 specs, 77+ unit tests, and CI on every push.

`Bash · OpenClaw · agent ops · cost governance`

---

### [Adapta](https://github.com/yielab/adapta)
*Self-hosted RAG + LoRA fine-tuning — your data stays in your infrastructure*

Most self-hosted RAG tools handle knowledge (retrieve from your docs). Most fine-tuning tools handle behavior (train tone and format). Adapta does both, composed on one OpenAI-compatible endpoint. An eval gate means an adapter never serves production unless it beats the base model on a held-out split — you can't accidentally deploy a regression. No telemetry. No cloud. Your embeddings, vectors, and model weights stay on your hardware.

`Python · FastAPI · SQLAlchemy · QLoRA · ChromaDB · PostgreSQL`

## Engineering approach

All three projects share the same discipline:

- **Spec before code** — every command and API endpoint has a written spec using RFC 2119 keywords (`MUST`, `SHOULD`, `MAY`) before implementation starts
- **Honest status tables** — READMEs say what works, what's in progress, and what isn't built yet
- **Real test coverage** — unit tests, integration tests, contract tests (schemathesis), and evals where relevant
- **CI on every push** — nothing merges without passing the test suite
- **No inflated claims** — metrics in the docs are generated by the code, not written by hand
